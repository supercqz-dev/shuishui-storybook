# UX 审计 & 设计方向（2026-05-27 晚）

## 审计前提

项目分两个用户群、两套体验：

| 入口 | 主用户 | 风格定位 |
|---|---|---|
| `/`（图书馆首页）+ `/books/[id]`（阅读器）| 3.5 岁水水 + 妈妈陪读 | **童趣、温暖、大触摸区、最小决策** |
| `/editor` + `/characters`（创作器） | 我（爸爸） | **工具感、信息密度合理、操作高效** |

混在同一个仓库里没问题，**色板和组件库分两套**就行。

---

## 1. 现有交互问题清单

### 1.1 图书馆首页（`/`）—— 儿童端但表现像后台

| # | 问题 | 现状 | 影响 |
|---|---|---|---|
| L1 | 没有书架氛围 | 灰白底 + 文字标题"水水的绘本 📚"，2-3 列 grid | 看着像 Notion 模板，水水不会觉得"我的书" |
| L2 | 头部空荡 | 只有标题 + 副标题 + 角色管理小链接 | banner 区缺失，需求里也指出要补 |
| L3 | 创作入口跟阅读入口混杂 | "+ 新建一本"是给爸爸的；"已出版"列表给水水。混在一个页面 | 水水可能会点错 |
| L4 | 书卡用粉色色块 + 5xl 🐰 兜底 | demo book 已经有图，但 cover_image 字段没填，bookId 链接的图是 page-01 没有标题 | 视觉粗糙 |
| L5 | 编辑/删除按钮跟阅读入口同色权重 | 三个按钮挤一行 | 对水水来说所有按钮一样诱惑——会乱点删除 |

### 1.2 阅读器 `/books/[id]` —— 整体 OK 但有小问题

| # | 问题 | 现状 |
|---|---|---|
| R1 | 翻页只能左右滑 + 点中间 | 桌面端没有键盘左右键支持 |
| R2 | 第一次打开没有"开始阅读"提示 | 直接显示第一页文字，孩子不知道能滑 |
| R3 | 文字 5 秒自动隐 | `chromeTimer` 自动隐藏，但没有显式触发再显示的引导 |
| R4 | 退出回首页要点左上角小箭头 | 触摸区太小，孩子按不到 |
| R5 | 末页之后没有"读完了"反馈 | 翻到最后一页就停，没有结束页 |

### 1.3 创作器 `/editor` —— 工具但工具感不足

| # | 问题 | 现状 | 影响 |
|---|---|---|---|
| E1 | form 阶段只有 2 个 textarea + 角色 grid | 没有引导提示 / 案例 / 历史 trigger | 我每次要从空白想 trigger |
| E2 | 8 页 review 一字纵向铺开 | 整页滚动，看不到全局 | 难以判断节奏（哪几页 close-up 集中、哪页缺图） |
| E3 | 单页生成时其他页全 disable | `anyRendering` 锁住所有按钮 | 不能并行生成多页（API 实际支持，UI 卡死） |
| E4 | 生成失败的错误信息只显示在顶部红框 | 没有针对性提示是哪页失败 | 失败页要滚回去找 |
| E5 | 编辑现有 book（F3 新加）没有"刷新当前页"按钮区分新画面 | 改了文字后再点生成会改图，但没有视觉提示"草稿状态" | 不清楚改了什么没保存 |
| E6 | 状态反馈弱 | "生成中…（约 1 分钟）"没有进度条；"保存中…"没有 spinner | 看着像卡死 |

### 1.4 角色管理 `/characters` —— 新加但有缝

| # | 问题 |
|---|---|
| C1 | 4 个角色的 yaml 字段差异大（papa 有 13 个字段，shuishui 有 8 个），编辑表单只暴露 5 个核心，其他塞进"高级 yaml"——非技术用户不友好 |
| C2 | 没有"预览 prompt 完整结构"——改 anchor 时不知道最终怎么拼到 image API 的 prompt 里 |
| C3 | 重生定妆图按钮（🎨）没有进度提示，3 分钟用户以为卡了 |
| C4 | 删除按钮跟编辑按钮同列，水水主角不能删但其他角色可——没有"草稿/已发布"概念，删了就没了 |

### 1.5 跨场景缺失

| # | 缺什么 | 备注 |
|---|---|---|
| X1 | 封面（cover_image）自动生成 | 要做：每本生成时一并出封面 |
| X2 | 图书馆 banner | 用户明确要求 |
| X3 | "草稿" 状态 | book schema 有 status 字段但 UI 没读 |
| X4 | 失败重试机制集中 | demo-story.mjs 有 retry，但 UI 单页生成要手动按按钮 |
| X5 | 阅读历史 / 收藏 | 暂不做（PRD §6 排除） |

## 2. 视觉设计语言

### 2.1 儿童端（图书馆 + 阅读器）

**关键词**：温暖、毛绒、绘本、不刺眼、奶油色、圆润

**色板**

| Token | Hex | 用途 |
|---|---|---|
| `cream-50`(底) | `#FFF8F0` | 主背景 |
| `cream-100`| `#FFEEDC` | 卡片底 |
| `pink-100` | `#FFE0EA` | 高亮区域 |
| `pink-300`(主) | `#FFA8C2` | 按钮 / 强调 |
| `pink-500` | `#FF6E9C` | hover / 重点 |
| `yellow-200`| `#FFE8A8` | 装饰、第二色 |
| `brown-700`(文字) | `#5C4434` | 主文字 |
| `brown-500` | `#8B6F5A` | 次文字 |

**字体**
- 中文：`'Noto Sans SC', 'PingFang SC', 'HarmonyOS Sans SC'`，权重 `500/700`，**字号大**：标题 32-48px，正文 18-22px
- 数字 / 英文：`'Quicksand', 'Comfortaa'`（圆体）

**圆角**：`rounded-3xl`（24px）打底，按钮 `rounded-full`，卡片 `rounded-2xl`

**阴影**：柔和漫射，不要硬阴影。`shadow-[0_4px_20px_rgba(255,168,194,0.15)]`

**动效**：缓动 300-400ms，带 spring 回弹。书卡 hover 微抬起（`translate-y-[-2px]`），按下 `scale-95`。

### 2.2 工具端（创作器 + 角色管理）

**关键词**：清晰、紧凑、专业、低饱和、信息密度

**色板**

| Token | Hex | 用途 |
|---|---|---|
| `gray-50`(底) | `#FAFAFA` | 主背景 |
| `white` | `#FFFFFF` | 卡片 |
| `gray-200` | `#E5E7EB` | 边框 |
| `gray-700`(文字) | `#374151` | 主文字 |
| `gray-500` | `#6B7280` | 次文字 |
| `pink-500`(强调) | `#EC4899` | 主按钮 / 链接 |
| `purple-500`(辅) | `#A78BFA` | 编辑/草稿状态 |
| `green-500` | `#10B981` | 成功 |
| `red-500` | `#EF4444` | 删除 / 错误 |

**字体**
- 中文：`'PingFang SC'`，权重 `400/600`
- 英文：`'Inter'`
- 字号：标题 24-32px，正文 14-16px，code 13px mono

**圆角**：`rounded-lg`/`rounded-xl`（8-12px），不要太圆

**阴影**：`shadow-sm` / `shadow`，专业感

**动效**：150-200ms 直线 transition，无 spring

## 3. 设计动作清单（今晚要做）

### 3.1 图书馆首页重做
- 顶部 banner（用 gpt-image-2 生成的图，回退方案：CSS 渐变 + emoji）
- 创作入口移到右上角 "+" 浮动按钮 / 隐藏菜单（水水不容易点到）
- 书卡：大圆角、cover 用 page-01 兜底+ ribbon "已出版"、标题压在底部半透明渐变
- 编辑/删除按钮收进每张卡的 "···" 菜单（hover 显示 / 长按显示）—— **不行**，用户说不要长按
  → 用专门的"管理"开关：进入管理模式后才显示编辑/删除（默认看到的是干净书架）

### 3.2 阅读器优化
- 加键盘左右翻页
- 加首页"轻轻滑屏开始阅读"提示（首次打开 3 秒后淡出）
- 末页加"读完啦 ✨"庆祝页 + 回首页大按钮
- 退出按钮放大到 56×56px

### 3.3 创作器优化
- form 阶段加"灵感"chip（点 chip 自动填 trigger，比如"今天不爱吃菜""第一次自己穿鞋"）
- review 阶段加左侧 8 页缩略图导航（点击跳转）+ 顶部 progress bar（生成进度）
- 单页生成不再 disable 其他页
- 失败错误显示在对应页卡，不是顶部
- 编辑现有 book 时：改了内容显示橙色"未保存"标记

### 3.4 角色管理
- 重生按钮加 spinner / 进度计时
- 列表卡片左下角加"上次更新"时间戳

### 3.5 全局
- 新增 banner/cover 集成
- 中文角色实验产物收下（如果效果好）

## 4. 取舍与今晚不做

- **不做**：暗色模式（孩子睡前用的话浅色更柔和，不必反色）
- **不做**：阅读统计、收藏夹、分享
- **不做**：换肤皮肤系统（一套就好）
- **不做**：动画过多（孩子注意力会被吸走，纯阅读体验为主）
